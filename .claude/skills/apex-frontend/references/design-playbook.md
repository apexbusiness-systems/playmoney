# Design Playbook

## Failures (❌)

- Designing UI before defining the job → pretty screens that don't convert.
- No state design → breaks under real conditions.
- Gesture-only interactions → low discoverability.

## Correct Pattern (✅)

Job → Flow → IA → Wireframes → Visual system → Prototype → Spec → Verify.

## Executable: 75-minute sprint

1. Job story + success metric
2. Primary flow (verbs)
3. State matrix per step
4. Thumb-first wireframes
5. Tokenized UI (type/space/color roles)
6. Micro-interactions (feedback, undo)
7. Run 5-task test; patch top 3 confusions

## Deliverables

- Flow diagram
- Screen spec(s)
- Copy deck (errors/empty/success)
- State matrix
- A11y notes
